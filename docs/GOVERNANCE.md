# reason:// Governance Model (v0.1)

**Status**: Bootstrap phase (2026–2027) → Neutral standards body

## Principles
- Meritocratic: Only artifacts that win live WARF arbitration enter the registry.
- Quality-gated: Only artifacts that pass the kappa > 1.15 threshold enter the registry.
- Neutral & open: No single entity owns the protocol long-term.
- Transparent: Every deposit, arbitration, and score is publicly auditable.

## Phase 1 — Bootstrap (now → 12 months)
- Registry operator: Astrognosy AI (explicitly stated in patent and spec).
- Namespace minting: First-come, first-served via `reason.register()`.
- Collision arbitration: Handled by WARF scoring node itself.
- Public audit log: All deposits, arbitration events, and scores are publicly accessible via SHA-256 chained audit records on the Xport node.

## Phase 2 — Transition (Q4 2026)
- IETF Working Group formed (draft already in progress).
- Registry handed to neutral foundation or IANA-equivalent body.
- Governance board: 1/3 operators, 1/3 researchers, 1/3 adopters (rotating).

## Phase 3 — Community (2027+)
- DAO-style proposals for new top-level namespaces (`reason://defense/...` etc.).
- Economic layer (provenance chains + proportional compensation per patent Claims 37-44) activated only after adoption threshold (e.g., 100 validated artifacts + 10k monthly resolutions).

This model mirrors DNS/HTTP: temporary central control with explicit, time-bound off-ramp to open standards.
